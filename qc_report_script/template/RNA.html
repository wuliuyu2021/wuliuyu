1 概述

转录组 ( Transcriptome ) 是指在特定环境或生理条件下的一个细胞或一个细胞群中所有RNA分子的集合,包括信使 RNA ( mRNA )、核糖体 RNA ( rRNA )、转运 RNA ( tRNA )及非编码 RNA ( ncRNA )。它有时被用来指所有的 RNA，或仅指 mRNA，指何种 RNA 取决于我们设计的特定的实验，以下转录组分析仅基于 mRNA 分析。转录组不同于正常基因组的稳定性，转录组会随外界环境的变化而变化。转录组可以反映不同实验条件和不同时间里基因表达的变化，但是除了转录衰减等 mRNA 降解现象外，因为无法获取其序列信息。根据实验设计获得不同的实验样本后，提取 mRNA 并通过二代测序获取 RNA 序列信息用于后续信息分析，本报告分析主要以转录组的差异基因分析为重点，并进行功能注释来进一步了解不同的实验设计或环境是如何对样本的各基因功能表达的表达进行影响。

2.1.1 RNA 样品

海普洛斯对 DNA 样品的检测主要包括以下几点：

  • 1) NanoDrop™ One/OneC : 检测 RNA 纯度 (OD260/280,OD260/230比值 )
  • 2) Life Invitrogen Qubit® 3.0 荧光定量仪 : 采用 Qubit™ RNA HS Assay Kit 精确定量
  • 3) Agilent 4200 TapeStation 系统：精确检测 RNA 完整性 ( RIN值 )

    • 2.1.2 文库构建及库检

    mRNA 捕获建库方案 : 真核生物大部分 mRNA 具有 polyA 的结构，因此我们可以采用带 Oligo ( DT ) 的磁珠将含有 polyA 结构的 mRNA 捕获出来，之后以片段化的mRNA为模版，随机寡核苷酸为引物，在M-MuLV逆转录酶体系中合成cDNA第一条链，随后用RNaseH降解RNA链，并在DNA polymerase I 体系下,以dNTPs为原料合成cDNA第二条链。纯化后的双链cDNA经过末端修复、加A尾并连接测序接头，用AMPure XP beads筛选200bp左右的cDNA，进行PCR扩增并再次使用AMPure XP beads纯化PCR产物，最终获得文库。

    图 2.1.1   RNA-seq 捕获建库流程图

    库检主要包括以下几点:

  • 1Kapa qPcr 定量试剂盒 : 精确定量文库浓度
  • Agilent 4200 TapeStation 系统 : 精确检测文库片段大小。

    • 2.1.3 上机测序

    库检合格后，把不同文库按照有效浓度及目标下机数据量的需求 pooling 后进行 Illumina HiSeq 测序。

    2.2 信息分析流程

    我们的信息分析流程主要包括数据质控，参考基因组比对，基因表达量分析，差异富集分析、变异检测分析、差异变剪切以及差异基因的蛋白互作分析。

    1）数据质控 ：测序数据的质量是下游信息分析结果是否可靠的重要前提，因此对于下机数据我们需要进行一定的质控来获取高质量的测序数据。

    2）参考基因组比对 ：从各样品质控后的 Clean Data 出发，将 Clean Data 比对到参考基因组进行比对结果的统计。

    3）基因表达量分析 ：根据比对结果进行基因定量和差异基因分析，基因定量主要是查看各转录本有多少测序序列覆盖 ( Reads Count )。

    4）差异基因分析和功能富集分析 ：通过基因定量信息进行样本间差异基因的分析，是下游分析的重点，最后根据在样本间获取的差异基因在通过功能注释库，比如 GO 和 KEGG 等数据库来获取差异基因显著富集的功能信息，以及差异基因的蛋白互作分析。

    5）变异检测分析 ：基于转录组数据进行 SNP 、Indel 、融合基因的变异检测以及注释，在转录组层面挖掘与疾病相关的变异信息。

    3 分析结果

    3.1 测序数据预处理

    测序获得的原始数据 (Raw Data) 存在一定比例低质量数据，会对后续信息分析造成很大干扰。合格的测序数据是后续信息分析结果是否可靠的前提，因此需要对原始数据进行数据的预处理和评估。我们通过使用自主研发的开源软件 fastp [1]对原始数据进行处理，对低质量序列进行过滤，过滤内容包括如下部分：

  • 去除带接头 ( Adapter ) Reads 序列
  • 去除 N ( N 表示无法确定碱基信息 ) 碱基数达到一定比例的 Reads (默认 5bp)
  • 去除所含低质量碱基 (质量值 ≤20) 超过一定比例 (默认 40%) 的 Reads

    • 上述的处理步骤均是对 Read 1 和 Read 2 进行操作。各样本测序数据预处理统计结果见表 3.1，更多详细信息请点击QC_Report。

    表 3.1 数据预处理统计表

    上表说明：
    (1) Sample ：样品名
    (2) Raw Reads(G) ：统计每个原始数据文件的 Reads 数量
    (3) Clean Reads(G) ：统计每个过滤后的数据文件的 Reads 数量
    (4) Q20、Q30(%) ：分别计算 Phred 数值大于20、30的碱基占总体碱基的百分比
    (5) Error rate(%) ：样本的错误率
    (6) GC Content ：计算碱基 G 和 C 的总数量和占总的碱基数量的百分比

    3.2 参考基因组比对分析

    测序数据的比对我们选取 HISAT2 软件，全称为 Hierarchical Indexing for Spliced Alignment of Transcripts [3]。HISAT2 是 BWT 算法的改进版，在比对速度上更快，在系统资源占用上更少。Clean Reads 比对到参考基因组上，获取其在参考基因组上的位置信息和序列的特征信息，信息文件以 sam 格式( sequence alignment/map format ) 存储，序列比对格式标准，由sanger制定 [4]。再用 Samtools 对 BAM 文件进行排序、创建索引、统计比对结果、过滤重复 reads 等处理，从而得到 bam 格式的最终比对结果。在通过比对的 bam 文件对序列在参考基因组上的比对情况进行统计得到比对信息。

    3.3 表达量分析

    表达量分析是差异分析的前提，表达量分析的实质是根据各样本的比对结果统计不同基因的 Reads count 值，即比对到该基因的 Reads 数有多少,也是后续差异基因分析的基础信息，如果基因的表达量高，则其 Reads 覆盖数也会高。我们采用 featurecounts [5] 进行 Reads Cunt 计算,featurecounts 的统计特征值主要分为 features 和 meta-features，在图3.3中以实线表示。features 通常是对应着注释文件 ( GTF ) 的 Exon 区域，meta-features 通常是对应于注释文件的 Gene 区域。在 RNA-seq 的情况下，每个基因指其所有外显子的总和。

    该软件根据每个染色体的 features 的起始位置进行排序分为128 kb 的 bins。将连续的 features 中数量相等的分类成相同的模块 ( Blocks ) ，每一个 bin 中 Blocks 的数量与 bins 中 features 的数目的平方根相等。比对的 Read 先与基因组的 bins 进行比较，再与 bins 中的 Blocks 比较，最后与 Blocks 中的 features 进行比较，最终的 Read 的定位结果是定位到了第 i 个 bin 中的第一个 Blocks 中，fenturecounts 还有一个主要特点，对于一个配对的 Reads，如果配对 Reads 其中一条 Read 比对到 A 基因与两个 Reads 都比对到 B 基因，最后会将该 Reads 计入 B 基因中，而其他软件如htseq-count会将该种 Reads 过滤。另外，为了避免基因长度对 Reads 数的影响，后面可以采用FPKM值进行基因的定量比较。我们首先采用 featurecounts [5] 和基因组注释文件对各样本统计其基因的 Reads 支持数，并转化为相应的 FPKM 值用于分析样本间相关性分析和不同基因表达水平变化。

    3.3.2 FPKM 统计

    FPKM ( fragments per kilobaseof exon per million fragments mapped ) 是指每1百万个 Map 上的 Reads 中 Map 到外显子的每1 K 个碱基上的 Reads 个数，是 RNA-Seq 的双端测序的基因表达值的表示方式，是对基因的测序深度和基因的长度进行校正后得到的校正值，我们对 Read Count 进行 FPKM 值转换，并绘制 FPKM 箱线图,通过图形可以观察不同实验条件下的基因表达水平的变化。

    箱线图 ：横坐标为样品名称，纵坐标为 log10(FPKM+1)，是对五个统计量(至上而下分别为最大值，上四分位数，中值，下四分位数和最小值)的绘制。

    3.3.3 样本相关性

    统计学上相关性分析 ( Correlation Analysis ) 是指对两个或多个具备相关性的变量元素进行分析，从而衡量两个变量因素的相关密切程度。对于 RNA-Seq 实验来说，生物学重复的实验样本可以通过相关性分析查看生物学实验操作的变动不大，非生物学重复关系的样本可以通过相关性分析观察不同实验条件间相似性和差异程度。通过各样本的基因定量关系，绘制样本相关性热图，观察各样本间的关系。相关系数越接近1，说明样品间表达的模式相似度越高。在下图中颜色越深的表名相关性越强。

    3.4 差异基因分析

    差异分析 ( Difference analysis ) 是在一组样本中找出产生差异的原因以及其对造成差异的影响程度。差异基因分析就是寻找跟实验处理相关的功能基因或转录本。我们根据基因的定量分析信息进行差异基因分析。差异基因的分析可以帮助我们在不同的实验条件下对功能基因的筛选。对于有重复实验样本间的分析使用 DESeq2 [6] 软件进行分析，该软件是基于负二项分布，对 Reads Count 使用DESeq标准化后，通过估计样品的测序深度和分布离差来计算每个基因的 pvalue 值，根据 Benjamini-Hochberg (BH) 来计算 FDR。对于无重复的实验样本使用EdgeR [7]，该软件基于负二项分布理论，使用 TMM 标准化 Reads Count ，用经验Bayes估计和精确检验确定差异表达式计算每个基因的 pvalue 值，根据 Benjamini-Hochberg (BH) 来计算 FDR。对差异分析结果的过滤按照 pvalue 值小于0.05，FoldChange (根据软件模型进行计算得到的)绝对值大于2的标准。软件的分析步骤是首先对样本的 Read Count 进行标准化，在根据模型进行假设检验获取 pvalue 值以及 FoldChange，最后进行多重校正获取 FDR 值。

    3.5 差异基因功能富集分析

    3.5.1 GO功能富集分析

    GO ( Gene Ontology ) 是描述基因功能的综合性数据库。可分为分子功能 ( Molecular Function )，生物过程 ( biological process ) 和细胞组成 ( cellular component ) 三个部分。生物过程是由分子功能有序地组成的，具有多个步骤的一个过程。分子功能描述在分子生物学上的活性，如催化活性或结合活性。细胞组成指基因产物位于何种细胞器或基因产物组中。对挑选出的差异基因应用超几何分布算法分析差异基因的有富集的GO功能,根据算法返回的 pvalue 判断其富集的 GO 功能。通过差异基因的 GO 注释分析，寻找差异基因的GO分类的富集条目以及发现不同样品的差异基因与哪些基因功能的改变有关。

    3.6 差异基因蛋白互作网络分析

    蛋白互作网络 ( Protein-Protein Interaction Networks ，PPI ) 是蛋白通过彼此之间的相互作用构成，参与基因表达调节、生物信号传递、细胞周期调控及能量物质代谢等生命过程的各个环节。我们使用 STRING ( https://string-db.org/cgi/input.pl ) 蛋白质互作数据库中的互作关系进行差异基因的 PPI 分析，对结果文件可使用老师可使用 Cytoscape 进行可视化展示 [14] 针对性作图。

    3.7 差异可变剪接分析

    可变剪接 ( Alternative Splicing ) 一个 mRNA 前体通过不同的剪接方式 ( 选择不同的剪接位点 ) 产生不同的 mRNA 剪接异构体，是真核生物基因和蛋白质数量有较大差异的重要原因。可变剪接可分为 Skipped exon, Mutually exclusive exon， Alternative 5' splice site, Alternative 3' splice site, Alternative first exon, Alternative last exon 和 Retained intron 等7种类型。我们使用 ASD ( Alternative Splicing Detector ) [9] 软件进行分析样本间的差异可变剪接，可变剪接形式和软件检测流程原理如下图所示，主要根据 altered junction reads 和 altered coverage of AS-exons 计算可变剪接的差异显著性。

    3.8 融合基因分析

    融合基因 ( Fusion gene ) 是指两个基因的全部或一部分的序列相互融合为一个新的基因的过程，置于同一套调控序列（包括启动子、增 强子、核糖体结合序列、终止子等）控制之下。对于融合基因的检测我们使用 START-FUSION [10] 对样本进行融合基因的检测。START-FUSION 对样本的检测包括 SATR [11] 比对，STAR-Fusion.predict，STAR-Fusionfilter 以及 FusionInspector 校正，此外，STAR-Fusion 会预测融合基因的 CDS 序列，并进行 PFAM (Database of Protein Families) 注释。具体流程如下图3.8所示：
